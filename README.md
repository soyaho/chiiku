# claude-dev-template

どんな開発にも対応できる、**Claude で開発するためのワークフローだけが最初から
整っている**最基底テンプレート。対象領域（ゲーム・Web・CLI・ライブラリ…）も
技術スタックも規定しない。持っているのは「変更パイプライン・サブエージェント・
設計原則の台帳・還流ループ」という**開発の進め方の骨格**だけ。

> 系譜: proto_ae「星と翼の詩」→ html-tap-adventure-template（タップアドベンチャー特化）
> → html-game-template（ブラウザゲーム汎用）から、ゲーム要素も外して抽出した最基底層。
> **正本はこのリポジトリ**——派生プロジェクトからの還流（`/backflow`）はここに集まる。

## 入っているもの

| パス | 中身 |
|---|---|
| `CLAUDE.md` | 開発規約。変更パイプライン（設計→実装→検証緑→独立監査→台帳→還流）を強制し、**プロジェクト固有の規約を育てるスロット**（構造・検証コマンド・約束事）を持つ |
| `KNOWLEDGE.md` | 設計原則（開発一般に通じるものだけ）＋バグ台帳の器。**原則0＝測れないものは設計工程で守る** |
| `.claude/skills/` | `/change` `/feedback-fix` `/run-checks` `/backflow` |
| `.claude/agents/` | `implementer`（実装委任・Sonnet）/ `advisor`（相談役・上位モデル）/ `auditor`（クリティカル欠陥の独立監査）/ `verifier`（検証実行係） |
| `.claude/hooks/` | SessionStart フック（依存導入のスロット。Claude Code on the web で検証を即実行可能にする） |

## 入っていないもの（意図的に）

- **テストハーネスの実体** — 検証の器は領域ごとに違う（ブラウザゲームなら Playwright の
  実入力ハーネス、CLI なら E2E スクリプト、ライブラリなら公開APIテスト…）。
  テンプレートが持つのは「実インターフェース主義で検証する」という**契約**と、
  検証コマンドを書き込むスロットだけ。**立ち上げたら最初の実装より先に検証の器を作る**
- **スターター実装** — 何かのコードを置くと、それが領域を規定してしまうため

## 階層——どのテンプレートから立ち上げるか

```
claude-dev-template          … 開発一般（このリポジトリ）
└─ html-game-template        … ブラウザゲーム汎用（実入力ハーネス Playwright 込み）
   └─ html-tap-adventure-template … タップアドベンチャー特化（ジャンルの設計原則込み）
```

**作るものに一番近い層から立ち上げる**のが正しい（下の層ほどハーネス・原則・スターターが
具体的で、立ち上がりが速い）。ブラウザゲームを作るのに claude-dev-template から始めるのは
損——html-game-template を使う。ここから始めるのは、既存の層に当てはまらない開発
（ツール・サービス・ライブラリ等）のとき。

新しい領域を繰り返し作るようになったら、そのプロジェクトの `CLAUDE.md`「固有の約束事」・
検証の器・KNOWLEDGE.md の領域原則を、このテンプレートから派生する**新しい中間テンプレート**
に昇格させる（html-game-template がそうして生まれた）。

## 新しいプロジェクトの始め方

1. このテンプレートから新リポジトリを作る（GitHub の「Use this template」か clone）
2. セッションは**新プロジェクトのリポジトリで**始める
3. 最初の指示は繋ぎ方ではなく**仕様の言語化**に使う。例：

   > claude-dev-template から立ち上げてある。仕様：〇〇を作る。利用者は△△、
   > コアの価値は□□、クリティカル欠陥は◇◇。CLAUDE.md の規約に従い、
   > まず検証の器を整えてから、振る舞いを足すたびに検証も足して green を維持して。

4. CLAUDE.md 冒頭に仕様と「このプロジェクトの構造」を書き、**検証コマンドを整備**し、
   `.claude/hooks/session-start.sh` の依存導入を書き換える。
   `.claude/` と `KNOWLEDGE.md` の原則はそのまま持ち越す

## 運用の約束（境界の還流）

`.claude/` と `KNOWLEDGE.md` の設計原則は各プロジェクトに vendored コピーされる**写し**で、
正本はこのテンプレート。プロジェクトを進めると「これはワークフロー自体の改善だった／
開発一般の原則だった」という学びが必ず出る。**写しだけ直して終えず、`/backflow` で
このテンプレートに還流させる**。各テンプレートは自分の直接の親に還流する
（ゲーム → html-game-template → claude-dev-template）。
