# 検証実行

1. CLAUDE.md の「検証コマンド」節を読む。
   - **未整備（TODO のまま）なら実行より先に整備する**：実インターフェースを通す検証を
     1コマンドに束ね、CLAUDE.md に書き込み、`.claude/hooks/session-start.sh` の
     依存導入も書き換える。
2. 検証コマンドを実行する（初回は依存導入から）。
3. 結果の見方：
   - 全 green か、何がどう fail したか（テスト名・アサーション文言）。
   - fail の原因が「テスト側のバグ」か「成果物側の実バグ」かを区別する
     （実インターフェース主義が守られているか——内部直呼びのテストは証明力がない——も見る）。
4. 大量に回す・時間がかかる場合は `verifier` エージェントに委譲して結果要約だけ受け取ってもよい。
