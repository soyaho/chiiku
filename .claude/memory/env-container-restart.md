# 環境の癖: セッションコンテナは再起動で作業を失う

- **観測（2026-07-11）**: Claude Code on the web のセッションコンテナが再起動し、
  ①未プッシュのローカルコミット ②実行中の implementer サブエージェント（とその未保存編集）が消えた。
  リポジトリは古い状態で再クローンされることもある（git fetch で復旧可）
- **対策（このプロジェクトでの運用）**:
  - 準備コミット（仕様・契約・ハーネス）は作った**その場でプッシュ**する。
    open 中の PR への混入を避けたい場合は、PR の決着を待つのではなく
    そもそも PR を出すタイミングを実装完了後に寄せる
  - 長時間のサブエージェント実行中も、親文脈の成果物は随時コミット＆プッシュ
  - 復旧手順: `git fetch origin` → `git checkout -B <branch> origin/main` →
    文脈に残っている内容からファイル再生成 → 再委任
