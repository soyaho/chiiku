# てんびん — v0.2 差分仕様（テスターフィードバック反映）

v0.1（`tenbin_spec_v0.1.md`）への差分。記載のない項目は v0.1 のまま。

実プレイ観察: 子どもは喜んで遊んだ（コアは機能）。指摘4件を原則0（コア価値ゲート）に
通した結果が本差分。

## 1. 統一落下物理（v0.1 §4.1 の置き換え）

**v0.1 の「皿中心から半径120pxで放す→スナップ」を廃止。**
「つまんで皿の近くまで運ばないと乗らない」は物理世界の一貫性の破れ
（＝コアの価値「世界が答える」の破れ）だった。

- りんごを放すと**常に落下体**になる（指の速度を控えめに初速へ引き継ぐ。cap あり）
- 落下中、下向きに皿面を横切り、かつ皿の捕捉幅（皿の見た目より少し寛容）に入っていれば
  **皿に着地**（「ことん」＋その側の沈み込み）。皿の真上高くから落としても載る
- 皿に入らなければ地面へ（バウンドして静止。ペナルティなし。従来どおり拾い直せる）
- 皿は傾きで動くので、捕捉判定は毎ステップ現在の皿位置で行う
- ログ `dropOutside` の意味を「放して皿に載らなかった（地面に静止した）回数」に更新。
  カウントは**地面静止が確定した瞬間**（物理の結果が出た瞬間）

## 2. 無操作ヒント（ゴーストハンド）

「どうしたらOKか」の手がかり。文字・説明は使わず、**動詞（つまんで載せる）を世界がデモする**：

- その問題でまだ一度もりんごに触れておらず（firstTouch 前）、問題表示から
  **5秒（シミュレーション時間）**経過したら、半透明の手が「場のりんご1個 → 近い方の皿」
  へ動くループアニメを表示する
- どこかをタッチしたら即消え、再び5秒の無操作で再表示
- 手は描画のみ（ゲーム状態を変えない・りんごを動かさない）
- 「何を考えさせるか」（多い＝重い＝下がる）は完了演出（下がった側が食べる）が担う。
  それ以上の誘導はしない（正解の先回りは動詞体験を奪う）

## 3. BGM（WebAudio 合成）

- 静かなペンタトニックのアルペジオ＋低いパッドのループ。すべて合成（アセットなし）
- **物理音（答え）をマスクしない**: 音量は効果音より明確に小さく、完了演出中は自動で減衰
- 初回タッチで開始（AudioContext 制約）。生成失敗時もゲームは動く
- 親メニューに ON/OFF トグル。設定は `localStorage` キー `tenbin_prefs`（JSON、falsy 安全。
  例 `{"bgm":true}`）に保存し、リロード後も維持
- BGM の揺らぎはゲーム進行の乱数列とは**独立の固定シード RNG**を使う
  （ゲームの配置再現性を壊さない）

## 4. テーマバリエーション

学習対象（数量比較）は不変のまま、見た目をセッションごとに変える：

- **果物 3種**（例: りんご・みかん・もも）× **動物ペア 3種**（例: クマ&ウサギ・ネコ&ヒヨコ・
  パンダ&カエル）。すべて Canvas プリミティブ描画・絵文字不可・重さは全て同じ（=1個）
- セッション開始ごとにゲーム RNG から選択（同 `?seed` なら決定的）
- 「もう1回」で別テーマに出会えることが再開の楽しみになる

## 5. 読み取り面の追加（契約は test-hooks.md が正）

`window.__tenbin` に `theme` / `bgmEnabled` / `hintActive` を追加。
`menuRegions` に `bgm` 領域を追加。
