# chiiku — v0.3 差分仕様（ミニゲーム群の器＋第2ゲーム「わけっこ」）

v0.1/v0.2 への差分。記載のない項目は従来どおり。
実プレイ観察: コア価値（世界が答える手応え）は成立を確認。本差分は
①タイトル/メニュー画面（ミニゲーム群の器）②第2ゲーム「わけっこ」の追加。

## 1. 画面遷移（ミニゲーム群の器）

```
起動 → タイトル → メニュー → ゲーム（てんびん / わけっこ）
                     ↑            │ おうちボタン（0.8秒押し込み）
                     └────────────┘
```

- **タイトル**: 文字なし。ゆれる天秤と動物たちの絵＋やわらかい背景。どこでもタップ→メニュー
- **メニュー**: 絵タイル2枚（てんびん＝天秤の絵 / わけっこ＝2匹と分けた果物の絵）。
  タップでそのゲームを開始。文字・数字なし
- **おうちボタン**: ゲーム中の右上に小さな家の図形。**0.8秒（シミュレーション時間）押し込み
  続けると円ゲージが満ちてメニューへ戻る**。短いタップでは何も起きない（誤タップで
  進行が飛ばない）。確認ダイアログは作らない（✗なし・文字なし）
- 親メニュー（左上2秒長押し）・縦持ちブロックは全画面で有効
- セッションの開始＝ゲーム入場時（title/menu 滞在中はセッションなし）。
  おうちで離脱したセッションは completed:false のまま残る（従来の中断と同じ扱い）

## 2. 第2ゲーム「わけっこ」

**学習対象**: 数の対応づけ・分配（相手の欲しい量を読み取り、ちょうどに分けてあげる）。
テスター提案「2人のお腹のすき具合を見せて、ちょうどよく分けてあげる」をそのまま核にする。
原則は共通: 動詞が先（分けてあげる）・世界が答える（動物が食べる/静かに待つ）・✗を出さない。

- 画面: 左右に動物（テーマの動物ペアを流用）。各動物の前に皿。**頭上に思考泡＝
  欲しい果物のシルエット（輪郭のみ）を欲しい数だけ並べる**（数の個数表現。数字は使わない）
- 中央下に果物の山（総数 = 左右の欲しい数の合計）
- 操作は v0.2 の統一落下物理を流用（つまむ・運ぶ・落とす・皿が捕捉・可逆）。天秤はない（皿は固定）
- 皿に果物が載ると、その側の泡のシルエットが1つずつ実色に埋まる（対応づけの可視化）
- **完了判定**: 全果物が皿上＋左右の個数が欲しい数と一致＋1秒静止 → 両動物が同時に食べて
  満腹演出（おなかぽんぽん＋星パーティクル＋明るいアルペジオ）→ 1.5秒で次問へ
- **不一致で全部置いた場合**: 動物は食べずに**静かに待つ**。足りない側の思考泡がゆっくり
  ふわふわ拡縮する（明滅は1秒3回未満・✗やブザーは出さない）。移し替えはいつでも可逆
- レベルテーブル（データ駆動・5問。左右はシード乱数で反転）:
  ```js
  const WAKEKKO_LEVELS = [
    { a: 2, b: 1 }, { a: 2, b: 2 }, { a: 3, b: 2 }, { a: 3, b: 4 }, { a: 4, b: 3 },
  ];
  ```
- 進行ドット・セッション終了画面（星5つ＋手を振る）・タップでもう1回・無操作ヒント
  （ゴーストハンド: 山→泡の多い側の皿）・BGM・テーマは てんびんの部品/流儀を共通利用

## 3. ログ

- わけっこは `localStorage` キー **`wakekko_logs`**（配列・セッション単位）。スキーマは
  tenbin_logs と同型（sessionStart/completed/retried/problems[{level, firstTouchMs,
  durationMs, dropOutside, removedFromPlate}]）
- 親メニュー: 統計はゲーム別に表示（てんびん/わけっこ 各セッション数・完走率・もう1回率）。
  ログコピーは `{"tenbin":[...],"wakekko":[...]}` の1JSON。消去は両方消す
- 既存 `tenbin_logs` のスキーマ・互換は不変。破損耐性の流儀（falsy 安全・要素フィルタ）を踏襲

## 4. テスト入力の注入（1つ追加）

- **`?game=tenbin|wakekko`**: タイトル/メニューを飛ばして直接そのゲームに入る
  （テストの安定化・開発用。注入はこれで `?seed` / `?timescale` / `?game` の3つ）
- タイトル→メニュー→ゲームの実経路は、ハーネスのナビゲーションシナリオが実タップで通す

## 5. 読み取り面の追加（契約は test-hooks.md が正）

`window.__tenbin` に `screen` / `game` / `menuTiles` / `homeButton` / `wants` を追加。
既存 getter（apples/plates/counts/phase/problemLog/session 等）は わけっこ でも同じ意味で返す
（`balance` はてんびん専用のため わけっこ中は null）。
