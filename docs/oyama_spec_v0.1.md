# おやま — 仕様書 v0.1（第3ゲーム: 序列と合成）

保護者フィードバック「別のゲームを追加して」＋実プレイ観察「子どもが（おふねで）果物を
高く積み上げたがる」への応答。おふねの容量（あまりのあるわり算の核）は外せないため、
**「高く積む」という子どもが持ち込んだ目標に、専用の世界で応える**（原則9補記1）。
advisor 相談で候補3案から採用（却下理由は §7）。

## 1. 世界律（1本）

> **ものは、自分と同じか大きいものの上でだけ落ち着く。
> 大きすぎるものは、置いた瞬間にゆらっと傾いてすべり降りる。**

- 学ぶ数量演算: **序列**（大小比較の連鎖・推移律）と**合成**（部分から全体を立てる）。
  数量演算ゲート（tenbin_spec_v0.4 §1）に明記されつつ未カバーだった演算
- 動詞: **つみあげる**（てんびん=くらべる/たす/ひく、おふね=わける/はこぶ、と重複しない）

## 2. 画面と場

- 地面に**台座（岩）2〜3個**（シード乱数）。果物はここに積む（塔の基部）
- 果物のサイズは **5段階固定**（r = 18/24/30/37/45。隣接差6px以上＝6歳が見て判別できる差。
  テーマ果物の拡縮で描く）。個数はラウンドとともに成長する（§5）
- 動物はプレイ画面に**置かない**（置けば「必ず食べる」律が発生し、塔を壊す義務を負う）
- 進行ドット・おうちボタン・親メニュー・BGM・ゴーストハンド（果物→台座）は共通部を再利用

## 3. 律の解決（崩壊を作らない——最重要）

- 置く: 塔の最上段が自分と同じか大きい → **ゆらっと揺れて**着地（バネ-ダンパ・
  臨界減衰にしない。ハーネスが sway のオーバーシュート≥2 を検証）
- 大きすぎる: **いま置いた1個だけ**が静かに滑り降りて場に転がる（state 'field' に戻る・
  slideOffs++）。おふねの**あふれと同型**——即時・1個・喪失なし・すぐ拾える。
  塔全体の倒壊・遅延判定・安定メーターは実装しない（遅延した応答は罰に見える）
- 取り戻す: 塔のどの果物もつまんで下ろせる（可逆・takeBacks++）。**推移律により、
  どこを抜いても残りの塔は常に律を満たす**（非増加列から要素を除いても非増加）——
  抜いた上の果物は静かに沈下する（並べ直さない。台帳#22の流儀）
- 高さ: 塔の高さそのものが合成の連続量表示（おふねの乾舷と同格）

## 4. ラウンドと演出

- 1ラウンド = 場の果物を**全部どこかの塔に積む**＋全塔静止1秒で完了。
  塔の本数・形は自由（1本の高い塔・大きさ別の柱・双子の山——子どもが目標を持ち込める）
- 演出の序列: 置くたびのゆらっ ＜ ラウンド完了（星） ＜ **全部を1本の塔で完了（虹・最上位）**
  （おふねの「ぴったり満載」・てんびんの「釣り合い」と同格の、発見への特別応答）
- ドット5つで幕間 → リセット（既存の文法のまま）
- 完了は常に到達可能: 任意のサイズ多重集合は降順に積めば必ず1塔に全積みできる

## 5. 発見の曲線（v0.7 おおもの方式を踏襲。個数も成長する）

- ラウンド0: 4段階から**4個・全て異サイズ**（律の発見: 大は滑る・小は載る）
- ラウンド1: **5個・全5段階**（全て異サイズ）
- ラウンド2: 5段階＋**同サイズのふたご確定1組**＝6個（等値なら安定に積める＝第二の発見）
- ラウンド3以降: ふたご1〜2組（シード乱数）＝6〜7個
- 個数の成長＝積める塔の高さの成長が進行の新鮮さになる。
  1本塔には積み順の計画（降順ソートの身体化）が要り、後半も思考が残る

## 6. ログ・読み取り面

- `localStorage` キー **`oyama_logs`**（v1）:
  `{game:'oyama', v:1, sessionStart, endedBy, rounds:[{index, fruitsTotal, twins,
  towersAtClear, maxHeight, slideOffs, takeBacks, singleTowerClear, durationMs, endedBy}]}`
  読み込み正規化はキー基準（台帳#19）。親メニューの統計・コピー・消去に追加
- 読み取り面: `game:'oyama'`、`towers[]`（{x, y, sway, fruits:[id 下から順]}）、
  果物 state に `'tower'` を追加、`round` に上記カウンタのライブビュー、`?game=oyama`
- invariant: 果物総数保存＋**「支えより大きい果物が塔にない」**＋sway 非臨界減衰

## 7. 検討・却下した代替案（advisor 相談）

- **とんねる（合成分解）**: 隠す操作は数量を演算しない。見えない果物は「消えた」に見え、
  予想の照合先を外部提示なしに作れない
- **ならべっこ（序列ドリル）**: 正解序列が1つ＝世界が判定器化しドリル回帰（わけっこと同型）。
  序列の学びは「おやま」の支え律に吸収
- **タワー原案（重さでぐらつき→倒壊）**: 重さは見えず判断材料にならない。倒壊は蓄積の破壊＝
  実質の失敗演出。「大小＋いま置いた1個だけの滑り」に改めた

## 8. 実プレイの定点観測（原則0）

- slideOffs が序盤に出て減衰するか（律の発見曲線）
- singleTowerClear が自発的に出るか（1本塔という目標を自分で立てた証拠）
- ふたご初遭遇の発話（「おなじだ！」）
