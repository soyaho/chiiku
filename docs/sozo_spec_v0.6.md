# そうぞう — 差分仕様 v0.6（ブランド確立・置き場所の尊重・舟の手応え）

基底: `tenbin_spec_v0.1.md` ＋差分 v0.2〜v0.4、`ofune_spec_v0.1.md`。
この差分はユーザー（保護者）フィードバック4件への応答。変更はすべて
コアの価値「動詞が先・世界が答える・✗を出さない」の語彙で設計する。

## 1. ブランド「そうぞう」（フィードバック①）

複数ゲームを束ねる製品名を **そうぞう** とする（想像/創造 ＋ 音の中の「ゾウ」の言葉遊び）。

- `<title>` は「そうぞう」
- タイトル画面の主役は**ゾウのキャラクター**（Canvas プリミティブのみ。青灰色の丸い体・
  大きな耳・上向きの鼻・優しい目。呼吸と鼻のゆったりした揺れだけのアイドルアニメ）
- ロゴ文字「そうぞう」はタイトル画面のみ（文字許可領域）。ゲーム名（てんびん/おふね）と
  メニュータイルは変更しない——そうぞう＝棚、ゲーム＝中身、の二層を保つ
- ゲームプレイ画面にはゾウを出さない（数量演算ゲート: 装飾的応答を足さない）

## 2. 置いた場所にそのまま吸着（フィードバック②。両ゲーム）

グリッド自動整列（載せるたび全果物が整列座標へテレポート）を**廃止**する。
「子どもが置いた」という操作の結果を世界が尊重する——動詞が先・世界が答える。

- 載せる瞬間: 果物は**実際に置かれた水平位置**（皿/甲板中心からの相対X。端はクランプ）に吸着
- 縦方向は支えの物理で解決: 表面に置く。水平に重なる果物が既にあれば**その上に積む**
- 下ろす瞬間: 残る果物の位置は動かさない。**支えを失った果物だけ**が沈下する
- 皿の傾き・舟の喫水への追従は従来どおり（相対座標系は不変）

## 3. 舟は「静止していれば」載る・掴める（フィードバック③④）

### 3-1. 甲板捕捉は状態ラベルでなく実速度で判定する

帰航（returning）は流れの追従で桟橋へ漸近するため、「見た目は止まっているのに
状態は returning」の窓が数秒あり、そこに置いた果物が水に落ちていた（③の根本原因）。

- 捕捉条件を「**舟が実質静止している**こと」に変更: dock / unloading、または
  returning かつ実速度 < 閾値（30px/s・シミュ時間）
- sailing（前進中）の舟には従来どおり載らない——動く舟に載らないのは世界律として一貫

### 3-2. 舟はいつでも手で掴める（held）

- 舟状態に **held**（手に持たれている）を追加。dock だけでなく sailing / returning 中も
  掴める（unloading 中のみ不可＝荷下ろしと競合させない）。掴んだ瞬間に慣性は消える
- リリースは一本化: 強い（閾値以上）→ 出航（sailing）。弱い → 桟橋から離れていれば
  流れが戻す（returning）、桟橋なら dock

### 3-3. 甲板の果物は取り戻せる（可逆性）

- 舟が dock のとき、甲板上の果物は個別につまんで下ろせる（てんびんの「皿から下ろす」と対称。
  やり直しは常に静かに可能）
- 作用先の一意化: 甲板果物のタッチ判定は狭め（果物半径＋10px）、それ以外の船体タッチは
  舟のつかみになる。荷下ろしキューは消費時に果物が甲板上かを再検証する（防御）
