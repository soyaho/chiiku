# てんびん v0.4 差分仕様 — 自由遊び化（応答する世界）

v0.1〜v0.3 への差分。記載のない項目は従来どおり。

## 0. 診断（実プレイ第3回）

子どもはこの世界を「自分の目標を持ち込める物理世界」として信じ始めた
（同じ数にしたい・動物に食べさせたい）のに、ドリル構造（決められた問題・
全部載せたら強制終了）とスクリプト化された応答（わけっこの思考泡＝答えの提示）が
その信頼を跳ね返した。設計原則9を「操作の物理」から**目標の物理**
（子どもが世界に持ち込む意図にも世界律で応答する）へ拡張する。

## 1. 数量演算ゲート（応答の防波堤・最重要）

> **世界の新しい応答は、数量の演算であるものだけを許す**
> （皿に載せる＝足し算、食べさせる＝引き算、傾き＝比較）。
> 数量を変えも明かしもしない装飾的応答（なでると笑う、投げると芸をする等）は追加しない。

「創造性に応える」系の要求は今後すべてこのゲートを通す。
皿（加算・比較）と口（減算）で代数として閉じていることがこの設計の核。

## 2. てんびん自由遊び（ドリル廃止・ラウンド制）

- **LEVELS テーブル（5問ドリル）廃止**。ラウンド制:
  場の果物（総数4〜9・シード乱数・偶奇混在。難易度ランプは入れない）を
  **全部さばく（皿に載せる or 食べさせる）**＋天秤静止1秒でラウンド完了 →
  完了演出 → 新しい果物がころころ入って次ラウンドへ（1.5秒遷移）
- **食べさせ（引き算）**: 果物を動物の口ゾーンに放す/落とすと**必ず食べる**（拒食しない。
  「食べてくれない」は✗の変種）。「ぱくっ」＋喜びバウンス＋おなかが微小にふくらむ
  （上限あり・ラウンドでリセット。体の大きさを第二のカウンタにしない）。
  **皿の上の果物も食べさせ可**（皿から取る＝引き算が釣り合いの道具になる:
  奇数でも1個食べさせれば残りを等分できる）
- **落下・リリースの解決順**: 口ゾーン → 皿の捕捉帯 → 地面。
  口ゾーンと皿捕捉帯は**幾何的に非重複**にする（動物を外側へ寄せ、口ゾーン半径を調整。
  invariant としてハーネスが検証）
- **釣り合い祝福（瞬間の応答・何度でも）**: 両皿≥1個・水平（|角度|・|角速度|が閾値以下）が
  1.2秒（シミュ時間）継続 → 控えめな虹のきらめき＋柔らかいチャイム。
  **デバウンス**: 皿の構成（左右の個数の組）が変わるまで再発火しない（バネの揺れで連射しない）
- **演出の序列**（応答のインフレ防止）:
  釣り合いきらめき（小） ＜ ラウンド完了ファンファーレ（中: 差あり→下がった側が食べる／
  全部食べさせて皿空→両動物おなかぽんぽんの満腹バリアント） ＜
  **釣り合ったままラウンド完了（最上位: 虹＋特別ファンファーレ。v0.1「同数＝特別」の思想の頂点）**
- **進行ドット＝ラウンドカウンタ**: 完了ごとに1つ点灯。5つ満ちたら**幕間**
  （星5つ＋動物が手を振る、約3.5秒・自動で戻る）→ ドットをリセットして継続。
  ゲームは終わりを押し付けない（終わりはおうちボタン/放置＝子どもと親が決める）
- 無操作ヒント（ゴーストハンド）・テーマ・BGM・親メニューは従来どおり

## 3. タイトル・メニューの文字解禁

- **文字（ひらがな中心）はタイトル・メニュー・親メニューで使用可**。
  **ゲームプレイ画面内は従来どおり文字・数字禁止**（世界が答える原則のため）
- タイトル: 大きな天秤の絵（主）＋「てんびん」ロゴ（従）。ゲーム画面と明確に区別が
  つく構図（暗めの背景・中央ロゴ等）
- メニュー: てんびんのタイル1枚（絵＋「てんびん」ラベル）。将来のゲーム枠は
  空タイルを置かず、追加時にタイルを足す

## 4. わけっこ引退

- ゲームとしては削除（メニューから撤去・コードパス削除）
- **`wakekko_logs` の保存データは消さない**: 親メニューのログコピー JSON に
  legacy として含め続け、ログ消去の対象にも含める（ログ喪失させない）
- `?game=wakekko` はパラメータとして受けて **tenbin にフォールバック**（falsy 安全）
- 却下した代替案（動的空腹の手直し）と理由は KNOWLEDGE.md 棚卸し記録に記載

## 5. ログ v2（`tenbin_logs`）

```js
{
  sessionStart: <ISO文字列>,
  game: 'tenbin', v: 2,
  endedBy: 'home' | undefined,   // 離脱手段（タブ閉じ等は undefined のまま）
  rounds: [
    {
      fruitsTotal: <数値>,
      fedL: <数値>, fedR: <数値>,          // 食べさせた数（引き算の使用）
      balanceCelebrations: <数値>,          // 釣り合い祝福の発火回数（デバウンス後）
      clearedBalanced: <bool>,              // 釣り合ったまま完了したか
      dropOutside: <数値>, removedFromPlate: <数値>,
      durationMs: <数値>,
      endedBy: 'cleared' | 'home'           // 完了 or 途中離脱（部分ラウンドも記録）
    }
  ]
}
```

- 旧形式（v0.1〜0.3 の `problems` 形式）と混在しても、読み込み・親メニュー統計・
  コピーは falsy 安全に動く（旧レコードは legacy としてそのまま保持）
- カウンタは操作/判定の瞬間に確定。永続化はセッション開始・ラウンド確定・離脱の瞬間
- 親メニュー統計（案）: セッション数・総ラウンド数・釣り合い達成回数・食べさせ回数

## 6. 読み取り面の変更（契約は test-hooks.md が正）

- 廃止: `wants` / `levels` / `problemIndex` / `problemLog`（ドリル・わけっこ由来）
- 追加: `mouths` {L:{x,y,r}, R:{x,y,r}}、`round`（進行中ラウンドのライブビュー）、
  `plates.catchHalfW`（非重複 invariant 検証用）
- `phase`: 'play' | 'celebrate' | 'transition' | 'interlude'
- `celebrationType`: 'left' | 'right' | 'balance' | 'fed' | null
