# おふね — 仕様書 v0.1（第2ゲーム）

chiiku の第2ゲーム。てんびん（触感の基準器・v0.4 自由遊び）の原則
（動詞が先・世界が答える・✗を出さない・数量演算ゲート・答えを提示しない）を継承する。

**学習対象**: 分割と反復——「7個を容量4の舟でどう運ぶか」＝**あまりのあるわり算の身体感覚**。
容量そのものも教えられず、**あふれの物理から子どもが発見する**。
てんびんが二項比較（静的）なのに対し、おふねは分割と計画（動的・時系列）。
「多い＝重い」の身体感覚が別の文脈（沈む・押し出しが重い）で再登場する＝転移の検証。

## 1. 世界律（この1本から全応答を導出する）

> **水にはゆるい流れがあり、浮かぶものを左岸（手前）へ運ぶ。
> 軽いものは流れに従い、押し出された重い舟だけが勢いで渡りきる。**

1. **投げた/落ちた果物** → 水に落ちてぷかぷか浮き、流れで左岸へ戻って拾える（喪失なし。浮遊中もつかめる）
2. **積みすぎ（容量+1個目を積んだ瞬間）** → 舟が深く沈んで水があふれ、**その果物が浮き出して左岸へ流れ戻る**。
   舟は「容量超過の状態」を持たない（操作の瞬間に世界が解決）。
   **世界が引き算を実演する**——これが容量の発見メカニズム。喫水線・目盛りは描かない
3. **出航** → 舟をつかんで右へ**押し出す**（リリース速度で撃進・水の抵抗で減速。タップ出航は作らない）。
   **重いほど初動が重く渡りが遅い**（積載量の手応え）
4. **帰航** → 対岸で動物が積み荷を食べ終えると、空になった舟は**流れが自然に左岸へ戻す**（ボタン・スクリプトなし）
5. **空/軽い舟を弱く押した場合** → 少し進み、流れに負けて戻ってくる（拒否ではなく物理）

## 2. 画面・遊び（論理座標 1024×768 横持ち）

- 左岸（画面左1/3）: 果物の山（ラウンドの供給。テーマの果物を流用）
- 中央: 水面（やわらかい波）。舟が左岸の桟橋に浮かぶ
- 右岸: テーマの動物2匹（口カプセル＝てんびんと同じ捕捉様式）
- **積む**: 果物を舟の甲板へ（既存の落下・捕捉物理と同型の甲板捕捉帯）。積むたび**喫水が連続量で下がる**
  （バネ-ダンパのぼよんぼよん。**臨界減衰にしない**——触感の継承）
- **直接食べさせも成立する**: 指で対岸まで運んで動物に放せば必ず食べる
  （「動物は必ず食べる」はシリーズの世界律。舟は強制ではなく魅力——一度に沢山・ぼよんぼよん・
  虹の航跡——で選ばせる）。右岸の動物以外の場所に放した果物は転がって水へ→流れで左岸へ戻る
- **ぴったり満載**（load == capacity）で渡りきった便は**虹の航跡**（子どもが実験で発見した容量に
  ぴったり合わせたことへの応答。てんびんの釣り合い祝福と同格の特別）
- 対岸到着: 動物が積み荷を1個ずつ食べる（ぱくっ・喜び）→ 空舟が流れで帰る
- **ラウンド**: 果物6〜10・容量3〜5（シード乱数。**総数%容量≠0 のラウンドが混ざるように生成**——
  あまりのある分割こそ新しい数学。難易度ランプなし）。全部食べさせたら完了演出 →
  新しい果物が転がり込む。進行ドット・5つで幕間・おうち/親メニュー/ヒント/BGM/テーマは てんびんの流儀を共通利用
- 完了演出の序列: 虹の航跡（便単位・何度でも） ＜ ラウンド完了 ＜ **全便ぴったり満載で完了（最上位）**
- ゴーストハンドのデモ: 果物→舟の甲板（＋舟を押し出す動き）

## 3. 数量演算ゲート照合（全応答が数量の演算に閉じる）

| 応答 | 演算 |
|---|---|
| 積む→喫水が下がる | 加算（連続量の表示） |
| あふれ→1個浮き戻る | 容量との比較＋引き算 |
| 押し出しの重さ・渡りの速さ | 積載量の測定 |
| 動物が食べる | 減算 |
| 虹の航跡 | 等価比較（load == capacity） |

装飾的応答（水鳥・魚・波の芸など数量を変えも明かしもしないもの）は追加しない。

## 4. 却下した設計（わけっこの教訓の適用。理由つき）

- **満載喫水線（Plimsoll line）** — 線＋虹の組は「マークされた目標状態に合わせると報酬」＝
  外から提示された目標。数字でなく線で示しているだけで思考泡と同型。乾舷（水面とふなべりの隙間）
  自体が連続量の表示器なので線は不要
- **積みすぎ→動けない舟** — 「正しい操作をしたのに何も起きない」（原則9が名指しする破れ）。
  あふれ物理に置き換えて状態ごと消した
- **タップ出航** — この世界の文法（つかむ・運ぶ・放す→物理が解決）への異物。コマンドUIの匂い
- **直接食べさせの拒否** — 「動物は必ず食べる」というシリーズ確立済みの律への背信
- **あふれを出航時に波でこぼす案** — 応答が操作（積む）から遅延し、押した罰に見える。
  あふれは積んだ瞬間に

## 5. ログ（`localStorage` キー `ofune_logs`）

```js
{
  sessionStart: <ISO文字列>, game: 'ofune', v: 1,
  endedBy: 'home' | undefined,
  rounds: [
    {
      fruitsTotal: <数値>, capacity: <数値>,
      trips: <数値>,            // 積載≥1で渡りきった便数
      exactFullTrips: <数値>,   // ぴったり満載で渡りきった便数
      overflowEvents: <数値>,   // あふれの発生回数（容量発見の学習曲線の材料）
      directFeeds: <数値>,      // 指で直接食べさせた数（舟が選ばれたかの材料）
      splashes: <数値>,         // 果物が水に落ちた回数
      durationMs: <数値>, endedBy: 'cleared' | 'home'
    }
  ]
}
```

- カウンタは操作/物理確定の瞬間に確定。永続化はセッション開始・ラウンド確定・離脱の瞬間
- 読み込みは falsy 安全＋数値正規化（台帳 #16 と同水準）。親メニューのコピー JSON・消去に
  `ofune_logs` を加える（tenbin/wakekko/ofune の3キー）

## 6. 定点観測（原則0。次の実プレイで見るもの）

- **容量を線なしで発見できたか**: overflowEvents がラウンド序盤に出て後半減るか
- **舟が選ばれたか**: trips vs directFeeds の比率
- 子どもが「あと何回で運べるか」を口にしたら分割の思考が起きている（親の観察）
