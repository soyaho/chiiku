# そうぞう — 差分仕様 v0.7（おおもの・投げの誠実さ）

基底: `tenbin_spec_v0.1.md` ＋差分 v0.2〜v0.4・`sozo_spec_v0.6.md`、`ofune_spec_v0.1.md`。
この差分は保護者フィードバック2件への応答。

## 1. 投げの誠実さ（フィードバック「船に載せられる高さの上限がある」）

**診断**: 容量ではなく入力系のバグだった。リリース速度の推定が「動かした間」の履歴だけを
見ていたため、**止めて狙ってから放しても直前のスワイプ速度で射出**され、高い位置から
放すほど滞空中に横へ流されて甲板を外れ水没していた（＝高さの上限に見える）。

**世界律**: 手を止めてから放したら、飛ばない（原則9）。
- 最後の移動から一定時間（180ms・シミュ時間）経って放したら速度ゼロ
- 速度の分母を「履歴の先頭〜今」に伸ばし、保持した時間ぶん自然減衰させる
- 両ゲーム・舟のリリースにも共通適用（「ゆっくり離す＝出航しない」がより誠実になる）

## 2. おおもの（フィードバック「回数の最後まで遊ぶ前に飽きる」）

**診断（advisor 相談）**: 飽きの根本原因は構造の反復ではなく**発見の枯渇**——
核の学び（傾き＝比較、あふれ＝容量）はラウンド1〜2で発見が完了し、以降は同じ演算の
反復になる。処方は装飾の鮮度ではなく**新しい数量現象の投入**。

**おおもの** = 重さ2の大きい果物（見た目約1.5倍・つかみ判定も拡大）。

- **てんびん**: 傾きが「個数の差」から**「重さ合計の差」**になる → 「少ないのに重い」
  （個数≠重さ）という第二の発見。釣り合いは重さ合計の一致で祝福（おおもの1個＝通常2個）。
  釣り合いデバウンスは両皿の**重み多重集合**をキーにする（構成が変われば別の釣り合い）
- **おふね**: おおものは**容量2ぶん**を占める（喫水が2段沈む）→ あまりのあるわり算の
  単位が非自明になる。あふれ・虹（ぴったり満載）の式は不変（load が重さ合計になるだけ）
- **食べさせ・荷下ろし**は従来どおり1口＝1個（重さ2が消えると傾き・喫水が大きく応答する
  こと自体が学び）

**出現曲線**（セッション内累計ラウンド・0始まり、シード乱数で決定的）:
- ラウンド 0〜1: 出現しない（基本律の学習曲線を守る）
- ラウンド 2: ちょうど1個確定（初サイクル内に発見の瞬間を保証する）
- ラウンド 3以降: 0〜min(2, 場の果物数/3) 個で漸増

**検討・却下した代替案**:
- 世界からのお願い（動物がほしい数をドット泡で提示）— 「外部提示の目標＋合致で報酬」は
  引退させた思考泡・満載喫水線と同型。復活させるなら連続量の身体表現に再設計してから
- サイクルごとのテーマ/時間帯変化 — 応答ではなく背景であり、発見の枯渇に効かない。
  おおもの定着後の低優先として保留

**ログ**: 両ゲームのラウンドレコードに `bigFruits`（falsy 安全・読み込み時数値正規化）。
実プレイの定点観測: ①ドット5つまで遊び続けたか ②おおもの初遭遇で発話（「おおきいから
おもい」「はいらない」）＝発見の証拠 ③ラウンド3以降の durationMs が減衰しないこと。
